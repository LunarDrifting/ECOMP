## Accomplishments

- Added canonical transition service:
  - New `src/services/state-transition.service.ts`
  - Exposes `canTransition(...)` and `assertTransitionAllowed(...)`
  - Canonical rejection error: `Illegal state transition`
  - Includes matrix handling for required flows and DONE idempotent no-op allowance
- Guarded service state mutations:
  - `markTaskDone` asserts `current -> DONE` transition before transactional mutation
  - Resolver unblocking path asserts `BLOCKED -> NOT_STARTED` before mutation
  - Instantiation blocked assignment path asserts `NOT_STARTED -> BLOCKED` in instantiation context
- Hardened db mutation helpers:
  - Replaced generic state update helpers with intent-scoped methods:
    - `setBlockedForInstantiation(...)`
    - `setNotStartedByIdsForUnblocking(...)`
    - `markDoneByIdForCompletion(...)`
- Added deterministic API mapping:
  - Completion route maps `Illegal state transition` to HTTP 409.
