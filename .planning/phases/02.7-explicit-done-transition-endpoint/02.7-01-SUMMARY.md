# Phase 2.7 Plan 01 Summary

**Added a canonical task completion path: markTaskDone service + POST /api/tasks/{id}/complete endpoint, with deterministic validation and dependency-resolution handoff.**

## Accomplishments
- Extended tenant-scoped db wrapper with DONE transition primitives:
  - single-task state update by tenant + task id
  - DONE state support in existing batch state update helper
- Added `markTaskDone({ tenantId, taskId })` service method:
  - validates task exists for tenant
  - rejects BLOCKED -> DONE attempts
  - handles already-DONE as idempotent no-op (`taskMarkedDone: false`, no mutation, no resolver call)
  - updates state to DONE and then calls `resolveDependenciesForTask`
  - returns `taskMarkedDone`, `tasksUnblocked`, `unblockedTaskIds`
- Added canonical endpoint: `POST /api/tasks/[id]/complete`:
  - parses `tenantId`
  - delegates to service
  - remains thin and Prisma-free
  - maps deterministic errors (404 not found, 409 blocked conflict)
