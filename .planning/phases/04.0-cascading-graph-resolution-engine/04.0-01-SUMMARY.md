## Accomplishments

- Added batched tenant-scoped helper methods in `tenantDb` for cascade resolution:
  - downstream edges by frontier task IDs
  - resolution task fields by task IDs
  - approvals with actor IDs for policy checks
  - PRECONDITION gates with condition payloads by task IDs
- Replaced single-hop resolver behavior with `resolveDependenciesCascade(...)`:
  - queue/visited traversal across runtime dependency graph
  - dependency checks based on runtime rows only
  - candidate gating via approval policy check (existing semantics) and PRECONDITION check-only evaluator
  - transition guard enforcement for `BLOCKED -> NOT_STARTED`
  - deterministic sorted `unblockedTaskIds`
  - deterministic status values: `resolved`, `noop_no_downstream`, `noop_no_eligible`
- Preserved transactional atomicity:
  - `markTaskDone` still performs pre-transaction validation checks first
  - DONE mutation and cascade resolver both execute inside one `prisma.$transaction`
  - no nested transactions introduced
- Extended integration tests:
  - multi-hop cascade test validates two-level traversal plus blocked preservation for incomplete upstream
  - rollback test injects mid-cascade failure using Vitest spy on tx-bound helper and asserts no persisted DONE/unblocks
