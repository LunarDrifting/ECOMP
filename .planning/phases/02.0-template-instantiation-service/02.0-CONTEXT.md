# Phase 2.0: Template Instantiation Service - Context

**Gathered:** 2026-02-27
**Status:** Ready for planning
**Source:** User-provided phase goal and constraints

<domain>
## Phase Boundary

Implement service-layer logic to instantiate Tasks for an ECO from a TemplateVersion, plus a new API endpoint to trigger it.

</domain>

<decisions>
## Implementation Decisions

### Locked Decisions
- Must follow `docs/ARCHITECTURE_CONTRACT.md`.
- No schema changes in this phase.
- Service-layer only.
- No route refactors except adding a new endpoint.
- Must use existing service pattern.
- Must not bypass `tenantDb` wrapper.
- Must not directly use `prisma` in route files.
- Must create `ECOPlan` record before Task creation.
- Must attach Tasks to `ECOPlan` and `ECO`.
- Must enforce tenant scoping on all queries.
- Must remain sequential (no parallel waves).

### Deliverables
- Service: `instantiateTemplateForEco(...)`
- API route: `POST /api/ecos/:id/instantiate`
- Verification via `curl`
- No modification of existing models

### Clarification for Current Schema
- Current `Task` model has `ecoId` but no `ecoPlanId`.
- In this phase, "attach Tasks to ECOPlan and ECO" is implemented via:
  - creating `ECOPlan` first for the ECO/template binding, then
  - creating Tasks scoped to the same ECO (`task.ecoId`), with ECOPlan inferred through 1:1 `ECOPlan.ecoId` relationship.

### Claude's Discretion
- Exact request payload and response shape for instantiate endpoint.
- Idempotency behavior when an ECO already has an ECOPlan binding.

</decisions>

<specifics>
## Specific Ideas

- Add tenant-scoped helper methods in `tenantDb` for ECO, TemplateVersion, ECOPlan, and Task operations needed by this flow.
- Keep route thin: validate request, invoke service, return JSON result.
- Use service-level transaction boundaries if needed while preserving tenant constraints.

</specifics>

<deferred>
## Deferred Ideas

- Schema migrations or model changes.
- Complex template task graph expansion beyond current schema capabilities.
- UI/client integration beyond curl verification.

</deferred>
