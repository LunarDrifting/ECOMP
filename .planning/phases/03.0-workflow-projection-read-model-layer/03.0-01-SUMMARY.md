# Phase 3.0 Plan 01 Summary

**Added a tenant-scoped, read-only workflow projection service and endpoint with batched reads and deterministic derived flags.**

## Accomplishments
- Added batched projection read helpers in tenantDb:
  - tasks for ECO (single query)
  - dependencies by taskIds (single query)
  - approvals by taskIds (single query)
  - gates by taskIds (single query)
- Added `getWorkflowProjection({ tenantId, ecoId })` read-model service:
  - loads all required runtime data under tenant scope
  - returns tasks with `state`, `approvalPolicy`, `hasGate`
  - returns dependencies with `fromTaskId`, `toTaskId`
  - computes derived flags exactly as required:
    - `isBlocked = task.state === "BLOCKED"`
    - `requiresApproval = task.approvalPolicy !== "NONE"`
    - `requiresPrecondition = exists PRECONDITION gate row for task`
  - performs no write operations and no business-rule re-evaluation
- Added thin Prisma-free endpoint: `GET /api/ecos/[id]/projection`.
