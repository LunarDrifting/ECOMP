# Phase 3.2 Plan 01 Summary

**Implemented deterministic tenant-scoped RBAC checks for task completion and approval creation with thin route-level 403 mapping.**

## Accomplishments

- Extended `tenantDb` with scoped auth helpers:
  - task owner-role lookup support (`ownerRoleId` in task reads + `findOwnerRoleById`)
  - approval create helper (`approval.createForTask`)
  - tenant-scoped actor role assignment lookup (`userRole.listRoleAssignmentsByUserId`)
- Updated `markTaskDone` authorization:
  - accepts `actorId`
  - allows completion only when actor has `ADMIN` role or actor role matches `task.ownerRoleId`
  - rejects unauthorized completion with deterministic Forbidden error
  - preserves existing BLOCKED rejection, already-DONE no-op, approval enforcement, and gate enforcement behavior
- Added `createApproval` service with owner-role authorization guard:
  - task must exist in tenant
  - actor must hold `task.ownerRoleId`
  - unauthorized attempts rejected with deterministic Forbidden error
- Updated routes:
  - `POST /api/tasks/{id}/complete` now requires `actorId` and maps Forbidden failures to 403
  - Added `POST /api/tasks/{id}/approvals` thin route delegating to approval service, with deterministic 403 mapping
