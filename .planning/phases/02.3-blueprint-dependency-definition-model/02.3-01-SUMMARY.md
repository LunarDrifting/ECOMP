**TemplateDependencyDefinition is now implemented as a tenant-scoped blueprint dependency model with explicit from/to relation naming and required indexes.**

## Accomplishments
- Added `TemplateDependencyDefinition` model with required fields: `templateVersionId`, `tenantId`, `fromDefinitionId`, `toDefinitionId`, `type`, `lagMinutes`, `createdAt`.
- Enforced explicit Prisma relation names for dependency endpoints:
  - `fromDefinition` -> `@relation("TemplateDependencyFrom")`
  - `toDefinition` -> `@relation("TemplateDependencyTo")`
- Added required inverse relations on `TemplateTaskDefinition`:
  - `outgoingDependencies TemplateDependencyDefinition[] @relation("TemplateDependencyFrom")`
  - `incomingDependencies TemplateDependencyDefinition[] @relation("TemplateDependencyTo")`
- Added required indexes: `@@index([tenantId])`, `@@index([fromDefinitionId])`, `@@index([toDefinitionId])`.
- Generated and applied Prisma migration `20260228025951_add_template_dependency_definition_model`.
