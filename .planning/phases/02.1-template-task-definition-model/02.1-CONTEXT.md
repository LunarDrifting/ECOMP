# Phase 2.1: TemplateTaskDefinition Model - Context

**Gathered:** 2026-02-27
**Status:** Ready for planning
**Source:** User-provided phase goal and constraints

<domain>
## Phase Boundary

Add Prisma schema support for `TemplateTaskDefinition` to represent blueprint task definitions under a `TemplateVersion`.

</domain>

<decisions>
## Implementation Decisions

### Locked Decisions
- Schema-only changes.
- Must follow `docs/ARCHITECTURE_CONTRACT.md`.
- `TemplateTaskDefinition` must include:
  - `templateVersionId` (required)
  - `tenantId` (required; references `Tenant.id`)
  - `parentDefinitionId` (nullable self-reference)
  - `name` (required)
  - `taskLevel` enum (reuse existing `TaskLevel`)
  - `ownerRoleId` (required)
  - `visibility` enum (reuse `TaskVisibility`)
  - `approvalPolicy` enum (reuse `ApprovalPolicy`)
  - `clockMode` enum (reuse `ClockMode`)
  - `createdAt`
- `TemplateTaskDefinition` must reference `TemplateVersion`.
- `TemplateTaskDefinition` must reference `Tenant`.
- `TemplateTaskDefinition.ownerRoleId` must explicitly reference `Role.id` with `onDelete: Restrict`.
- Must include required inverse relation on `TemplateVersion`.
- Must include self-referential parent/children relation.
- Must include `@@index([tenantId])`.
- Must include `@@index([ownerRoleId])`.
- No modification of existing model fields beyond required inverse relation additions.
- Sequential execution only.

### Claude's Discretion
- Additional additive indexes for definition lookup/performance.
- Relation naming (e.g., explicit self-relation name) to keep schema clarity.

</decisions>

<specifics>
## Specific Ideas

- Keep definitions append-only style for now with `createdAt` and no `updatedAt` unless required.
- Include indexes on `templateVersionId`, `ownerRoleId`, and `parentDefinitionId` if useful for traversal.
- Include required indexes on `tenantId` and `ownerRoleId`.
- Reuse existing enums exactly; do not create duplicate enum types.

</specifics>

<deferred>
## Deferred Ideas

- Service/API logic for instantiating full task trees from definitions.
- Validation/business rules for illegal definition graph transitions.
- Any refactor of existing models beyond required inverse addition on `TemplateVersion`.

</deferred>
