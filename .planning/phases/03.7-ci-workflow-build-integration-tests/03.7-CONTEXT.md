# Phase 3.7: CI Workflow - Build + Integration Tests - Context

**Gathered:** 2026-02-28
**Status:** Ready for planning
**Source:** User-provided phase goal and constraints

<domain>
## Phase Boundary

Add deterministic GitHub Actions CI that runs build plus integration tests on every push and pull request, using an isolated CI Postgres test database.

</domain>

<decisions>
## Implementation Decisions

### Locked Decisions
- No schema changes.
- No production behavior changes.
- CI-only scope (GitHub Actions workflow).
- Deterministic execution required.
- No secret leakage.
- No external service dependencies beyond CI-hosted Docker Postgres.
- Execution remains sequential.

### Required CI Behavior
- Add `.github/workflows/ci.yml`.
- Trigger on `push` and `pull_request`.
- Checkout repository.
- Setup Node (pin to 20 unless a stricter existing project version requirement is present).
- Install dependencies via `npm ci`.
- Start Postgres service container (`postgres:16`).
- Create test DB `ecomp_test_db`.
- Set `DATABASE_URL_TEST` to the test DB.
- Run `npx prisma migrate deploy` against `DATABASE_URL_TEST`.
- Run `npm run build`.
- Run `npm run test`.

### Safety Requirements
- Do not use `DATABASE_URL` in CI.
- Only set `DATABASE_URL_TEST` in CI.
- Preserve fail-fast behavior when `DATABASE_URL_TEST` is missing.

### Claude's Discretion
- Exact workflow step naming.
- Health-check and startup wait mechanics for Postgres.
- Optional npm cache usage under `actions/setup-node`.
- Optional minimal docs note if needed.

</decisions>

<specifics>
## Specific Ideas

- Use `services.postgres` with explicit healthcheck options.
- Keep build/test steps sequential in one job for deterministic ordering.
- Use per-step env for `DATABASE_URL_TEST` rather than global `DATABASE_URL`.
- Ensure Prisma migrate and tests both receive the same `DATABASE_URL_TEST`.

</specifics>

<deferred>
## Deferred Ideas

- Matrix builds across Node versions.
- Split jobs (build/test) with artifact passing.
- Additional lint/typecheck CI gates.

</deferred>
