- `emitAuditEvent({ tenantId, ecoId?, taskId?, actorId?, eventType, payload, tx? })` pattern

- derive ecoId from taskId if needed via tenant-scoped lookup

<task type="auto">
  <name>Task 2: Instrument write-side services with audit attempt/outcome events</name>
  <files>src/services/template-instantiation.service.ts, src/services/approval.service.ts</files>
  <action>
Emit required events with deterministic reason/status payloads:
- instantiate: ATTEMPT + SUCCESS/REJECTED
- complete: ATTEMPT + SUCCESS/REJECTED/NOOP
- cascade: CASCADE_RESOLVE summary
- gate precondition failure: GATE_PRECONDITION_FAILED

- tx-bound completion/cascade emission occurs inside same transaction client
- best-effort emission: catch failures and continue
- no behavior drift in route/service outcomes
- no payload PII/secrets/raw gate condition JSON
  </action>
  <verify>
    <automated>npm run lint -- src/services/template-instantiation.service.ts src/services/approval.service.ts || npm run lint</automated>
